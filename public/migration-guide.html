Swap API v1 â†’ v2 migration guide

Learn how to migrate to the latest version (v2) of Swap API

The Swap API has undergone a major update that will require updates in your integration. This new version offers:

  • a new set of 0x smart contracts that offer secure upgradability
  • increased security via the Permit2 contract for allowances
  • improved swap pricing
  • improved monetization features (trade surplus coming soon)

Migration to v2

Summary of changes

Here is a summary of changes to take into account when upgrading your integration from v1 â†’ v2 ðŸ‘‡

  • Urls have been updated
    • https://api.0x.org/swap/v1/price â†’ https://api.0x.org/swap/permit2/price
    • https://api.0x.org/swap/v1/quote â†’ https://api.0x.org/swap/permit2/quote
  • The chain is no longer designated by the url endpoint. You will need to use the new chainId query parameter in when making /price and /quote requests
    • https://polygon.api.0x.org/swap/v1/quote â†’ https://api.0x.org/swap/permit2/quote?chainId=137
  • The shape of the API responses has changed - some response parameters have been deprecated, while new ones were added.
  • The API response now includes a new issues object, which returns a list of validation issues that need to be resolved before the data can be submitted by the taker. As part of this update, we have removed the skipValidation field. Validation will always be performed, even if the taker has an insufficient balance or lacks the necessary allowance. All validation issues including those related to balance, allowance, and slippage will now be returned in the issues field.
  • For token allowances, instead of granting the approval to 0x Exchange Proxy, grant the Permit2 contract instead (0x000000000022d473030f116ddee9f6b43ac78ba3).
  • To submit a transaction to the network, upon receiving the /quote response, you will need (1) sign a Permit2 EIP-712 message from the /quote response, (2) inject the signature into the call data, and (3) send transaction to the network.

In v1

  1. Build price params
  1. Call /price
  1. Approve token allowance for 0x Exchange Proxy
  1. Build quote params
  1. Call /quote
  1. Send transaction to the network

In v2

  1. Build price params
  1. Call /price
  1. (New) Approve token allowance for Permit2
  1. Build quote params
  1. Call /quote
  1. (New) Sign Permit2 coupon from quote response
  1. (New) Inject signature into call data -> Send transaction to the network.

Detailed Migration Guide

Overview on how to update Swap API price and quote to handle the new v2 endpoints

  1. Get a 0x API key
  1. CallÂ /swap/permit2/price (code)
    1. Build required price params
      1. Add chainId as new param
      1. takerAddress has changed to taker
    1. In the response: sellTokenAddress changed to sellToken, buyTokenAddress changed to buyToken (code)
  1. Check approval for PERMIT2 to token allowance. Approve token allowance if not set already. (code)
  1. CallÂ /swap/permit2/quote (code)
    1. Build required quote params (price params + taker address)
      1. Add chainId as new param
      1. takerAddress has changed to taker
    1. In the responses: sellTokenAddress changed to sellToken, buyTokenAddress changed to buyToken (code)
  1. Sign the Permit2 EIP-712 message from /quote response
    1. signature = await signTypedDataAsync(swapQuoteV2.permit2.eip712); (code)
    1. Update PriceResponse and QuoteResponse types (code)
    1. Replace MAGIC_CALLDATA_STRING (code)
  1. Submit the transaction with permit2 signature
    1. UseÂ sendTransaction({account, gas, txOptions, data, value, chain) (code)
    1. Note that the /quote response structure has changed from v1 to v2 (to, data â†’ have moved under quoteResponse.transaction.to, quoteResponse.transaction.data) (code)
